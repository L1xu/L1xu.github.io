【R语言】PLS-PM路径分析教程（二）

发表于2020-05-20更新于2020-12-21

一、简介

pls-pm：Partial Least Squares Path Modeling，偏最小二乘路径模型

R包：‘plspm’

定义：

1.是结构方程模型的偏最小二乘方法。

2.是一种统计方法，用于研究观测变量和潜在变量之间的复杂多元关系

3.是一种数据分析方法，用于研究一组设定的显变量区块中，每个模块都能被潜变量和潜变量之间的相关性所代表。

在之前的文章中，已经介绍了‘plspm’包的安装和加载。以一组西班牙球队的获胜指数来展示案例。

代码如下：

PLS路径分析建模最常见应用之一是指数的计算，以量化一些重要概念，例如满意度指标，可用性指标，成功指标等，这些概念无法直接量化，但是可以使用其他一些指标例如以某种方式反映幸福或悲伤的程度，对这些信息进行汇总以建立满意度指数。同样的，通过有关西班牙队参加比赛的信息，然后将其汇总分析后可建立获胜指数。

潜在变量（Latent variables）：我们常常会遇到这样一种情况，即模型中的关注变量无法被观测或直接测量，例如满意度，动力，可用性和成功等无法直接衡量的概念。此外，在社会科学及行为科学（心理学，社会学，经济）中也存在许多类似的概念，生态学中，也会遇到诸如土壤肥力，栖息地结构等概念。

当我们考虑构建模型时候，我们会倾向于设想其之间预期因果关系，例如：营销经理提出新的退货政策以提高客户满意度；高中老师决定增加户外活动，以提高学生的学习成绩；教练制定训练计划，以提高球队的防守表现。潜在变量通常是假设变量；很难被观测或直接测量；被用作基础变量，有助于解释两个或多个可观察变量之间的关联；将多个变量汇总为更少因子的便捷方法。

与结构方程一样，我们通常根据某种理论来提出模型。针对西班牙球队获胜指数的例子，可以用一个基本但有用的结论来定义一个简单的模型。

the better the quality of the Attack, as well as the quality of the Defense,the more Success.

Minifest Variables：尽管我们无法直接量化潜在变量，但是我们可以通过可观测的变量来间接测量潜变量，这类变量通常称之为 Minifest Variables（MVs）,我们通常使用这些指标中包含的信息来获得潜在变量的近似表示。

同时，影响潜在变量的方式可以分为Formative Indicators 和 Reflective Indicators

假设一个研究人员尝试衡量球队的攻击模型，他可以使用两种方法：询问反应攻击的不同统计数据（Reflective Indicators ）；询问可能影响攻击的可能的因素（Formative Indicators）。

我们有攻击，防守，胜利三个潜在变量，现在需要分别为它们建立一套指标。

攻击：用得分指标来反应其攻击质量，对应数据[1:4]

防守：用失球相关指标反应其防守质量,对应数据[5:8]

胜利：用获胜场数等指标表示，对应数据[9:12]

内部模型（inner model ）：内部模型是模型的一部分，与潜在变量之间的关系有关。

外部模型（outer model ）：外部模型与每个潜在变量及其指标之间的关系有关。

二、模型构建

了解路径模型的基础后，尝试通过R语言‘plspm’包使用西班牙球队数据实现PLS-PM分析。

内部模型构建：可以将内部模型视为一个网络，并将其表示为矩阵格式，称之为路径矩阵。即plspm()中的path_matrix，是一个基本的R 矩阵，行和列的数目相同，对角线及其上方的元素必须为0，对角线以下的元素可以为0或1。

# rows of the inner model matrix
Attack = c(0, 0, 0)
Defense = c(0, 0, 0)
Success = c(1, 1, 0)
# path matrix created by row binding
# 通过 rbind() 来堆叠行向量
foot_path = rbind(Attack, Defense, Success)
# add column names (optional)
colnames(foot_path) = rownames(foot_path)
# let's see it
foot_path
##         Attack Defense Success
## Attack   0        0       0
## Defense  0        0       0
## Success  1        1       0
## 根据矩阵可以了解箭头的指向，看行-列如attack和defense
## 都是0说明没有任务元素指向它们，它们指向success
# plot the path matrix
innerplot(foot_path)

外部模型构建：

# define list of indicators: what variables are associated with
# what latent variables
foot_blocks = list(1:4, 5:8, 9:12)
## 列表中包含三个元素，每个列表块一个。每个元素都是索引的向量。
## 分别对应潜在变量attack，defense，success

Vector of modes：
默认情况下，plspm()假定潜变量的指标属于reflective mode，用字母“A”表示，度量的另一种模式是formative mode ，用字母“B”表示。建议明确定义Vector of modes，可以使用与潜在变量数量一致的字符串表示。

# all latent variables are measured in a reflective way
foot_modes = c("A", "A", "A")
# Success in formative mode B
foot_modes2 = c("A", "A", "B")

函数plspm()构建路径模型

# run plspm analysis
#plspm(Data, path matrix, blocks, modes = NULL)
foot_pls = plspm(spainfoot, foot_path, foot_blocks, modes = foot_modes)

路径分析建模完成。完整代码如下：

# load package plspm
library(plspm)
# load data spainfoot
data(spainfoot)
# first 5 rows of spainfoot
head(spainfoot, n = 5)

# rows of the path matrix
Attack = c(0, 0, 0)
Defense = c(0, 0, 0)
Success = c(1, 1, 0)
# path matrix (inner model)
foot_path = rbind(Attack, Defense, Success)
# add column names
colnames(foot_path) = rownames(foot_path)
# blocks of indicators (outer model)
foot_blocks = list(1:4, 5:8, 9:12)
# vector of modes (reflective)
foot_modes = c("A", "A", "A")
# run plspm analysis
foot_pls = plspm(spainfoot, foot_path, foot_blocks, modes = foot_modes)
