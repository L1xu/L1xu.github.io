【R语言】PLS-PM路径分析教程（一）

发表于2020-05-19更新于2020-12-17

近期学习PLS-PM路径分析，需要安装’plspm’包，传统的安装方法提示报错，于是转而尝试安装github上的开发版本。中间遇到了很多问题，做一记录。

1.问题

# install "plspm"
install.packages("plspm", destdir = "D:/R_Files/lib")
  Warning in install.packages :
  package ‘plspm’ is not available (for R version 4.0.0)

根据github上的文档提示，如以下代码进行安装。 

# install "devtools"
install.packages("devtools") 
library(devtools)
# install "plspm"
install_github("gastonstat/plspm")

安装时提示以下报错。

WARNING: Rtools is required to build R packages, but is not currently installed.
Please download and install Rtools custom from https://cran.r-project.org/bin/windows/Rtools/.

2.解决方案

报错提示需要安装Rtools，从R语言官网下载rtools40，安装过程中个人变更了安装目录，其余全默认安装。
配置rtools路径：

进入R studio 中，运行以下代码

writeLines('PATH="${RTOOLS40_HOME}\\usr\\bin;${PATH}"', con = "~/.Renviron")

关闭R studio，重新打开。
运行以下代码：

Sys.which("make")

看输出是不是类似”C:\rtools40\usr\bin\make.exe”，具体因rtools安装路径而异，只要输出不是空字符串就行。这就表明路径配置成功。
配置结束。
重新运行。

# install "plspm"
install_github("gastonstat/plspm")

安装成功。
